git指令大全

发表于 2016-06-04 | 分类于 学习笔记 |

Git是目前世界上最流行的分布式版本控制系统，在我们工作中有很多的公司都是基于GIT这个系统来协同开发的。也是开发者必备技能。
Git是基于linxu系统的，可以说Git bash是一个简单的linxu命令行窗口。它本身也定制了很多命令。
接下来一起来记录一下Git 的一些命令行语句。

注意： 本文并不是git教程，只是总结了命令行语句，方便以后开发时候查看

配置个人信息

  • 姓名： git config --global user.name " your name "
  • 个人邮箱： git config --global user.email " your email "
  • 删除错误配置： git config --global unset "the KEY which you want to delete"
  • 查看配置信息： git config --glogal --list

常规操作

初始化一个空的仓库 ：git init
  • 这个命令是初始化一个.git 文件夹。里面存储项目的各个版本信息。
查看当前项目的状态 ：git status
  • 这个命令可以查看当前项目的是否有文件改变，需要提交到暂存区或者仓库。
  • 也可以用命令 git status -s 查看简要的状态信息
将项目中的文件提交到暂存区： git add './filename.txt'
  • 可以将你想提交的文件提交到暂存区，也可以使用提交全部的命令
  • git add . 或者 git add *;
将暂存区的文件提交到仓库： git commit -m "此次提交的备注"
  • 也可以 git commit -m -c 不需要备注信息。不推荐使用。每一次提交最好都备注当次提交代码的简要信息。
查看日志：git log
  • 或者：git log --oneline ————查看简要日志
  • 或者：git log --graph ————查看图形化的日志信息，有分支情况。
  • 或者：git reflog ————可以查看回退版本的信息。 配合回退版本查看。

.gitignore 配置忽略文件;

创建忽略文件,使用git dash 支持的linus命令行： touch .gitignore
  • 忽略文件是让git 忽略指定的文件。让其不会提交到版本仓库中。
  • .gitignore 文本中指定的文件是相对于.gitignore 文件的相对路径。
  • 忽略某个文件夹: /node_module
  • 忽略某个文件：./css/my.css
  • 忽略某一类文件：./css/*.css
  • 忽略目录下的所有文件：./css/.
  • 忽略所有名为”node_module”的文件夹：node_module
  • 以”#”开头的为备注信息。# 这条信息是注释

版本回退

回退到上一次提交的版本状态： git reset --hard Head
  • 回退到上上次提交的版本状态： git reset --hard Head^^
  • 回退到第n次前提交的版本状态： git reset --hard Head~n
    回退到具体的版本状态：git reset --hard [版本号]
  • 版本号为每一个版本的hash值，可以通过查看日志获取。
撤销对某个未提交的文件的修改：git checkout 文件名/.;
  • 未提交的文件是指已经修改，但是没有add 过的文件。
  • “.” 是指所有文件。

分支

查看当前所有分支： git branch
创建一个名为’dev’的分支： git branch dev
切换到dev分支：git cheakout dev
  • 注意，要与撤销对某个未提交的文件修改命令区分，命令一样，只是checkout 带的不同参数。
  • 创建并且切换到dev分支：git checkout -b dev
在当前分支，合并dev分支： git merge dev
  • 一般是在主分支’master’ 合并分支。
  • 合并,并且删除dev 分支： git branch -d dev
对比不同分支：git diff 分支1 分支2
  • 或者 git difftool 分支1 分支2

解决冲突

  • 每一次的分支合并，都可能带来冲突，冲突是由于两个程序员都同时修改了同一个文件造成的。
    举例子说：如果master下创建了两个分支，分别是a分支，b分支，如果a分支中有人修改了a.js文件。并且合并了master和a分支。
    而此时，b分支也修改了a.js文件，准备合并master,与 b分支，此时merge合并之后。就会产生冲突。
  • 冲突的解决，需要两个人协商之后由程序员删除修改重复之后，再次提交就可以解决。
  • 冲突是不可避免的，但是一定要与修改冲突文件的同事协商修改。
  • 查看冲突文件 git diff
  • 注意，修改冲突文件之后，一定要提交到暂存区，再提交到仓库才能解决冲突。

远程操作

下载版本：git clone '仓库地址'
  • 下载远程仓库。
  • 如果只需要下载最新一次提交，不需要下载整个仓库，加上参数 ‘–depth’
remote 命令使用
  • 添加主机： git remote add '主机名称' '仓库地址'
  • 删除主机： git remote rm '主机名称'
  • 查看已添加的主机: git remote
  • 查看远程主机信息： git remote -v
  • 查看详细的主机信息：git remote show '主机名称'
pull 拉取 git pull '主机名称' master
  • 默认名称为origin
  • 从远程仓库pull新版本,也可以用远程地址代替主机： git pull '远程地址' master
push 提交到远程仓 git push '主机名称' master
  • 默认主机名称为origin
  • 再本地仓库向远程仓库提交新版本，也可以i用远程地址代替主机：git push '远程地址' master
