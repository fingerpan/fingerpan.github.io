SPA原理简单解析

发表于 2016-06-16 | 分类于 my Bullshit |

SPA（Single page web application）

javascript的发展，离不开Ajax的助攻，可以说ajax改变了javascript的发展轨迹，给前端领域带来很大的变革。ajax从简单的验证数据到现阶段用来实现动态加载页面。单页面的提出，也是对ajax的运用。

单页面的诞生，是为了解决两大突出痛点。

1.内部页面的每次跳转，都会重新加载重复的库和公共代码。耗费流量资源。

这个问题可以通过设置缓存，设置本地数据存储减少数据的请求加载，但是也无法避免重复编译基础代码这个问题，用户等待时间有限，减少用户等待时间才能为公司带来更大的收益。

2.如果使用纯ajax动态加载，无法解决浏览记录问题。

用户无意关闭页面后，重新打开后又要通过层层点击才能进入之前浏览的页面，用户如果将页面分享给朋友后，朋友打开却还是首页。这样子的用户体验确实很糟糕。

单页面的原理。

单页应用的主要原理，其实很简单，利用a标签的锚点来解决链接的变动，但是页面不跳转，同时利用获得变动的锚点，使用ajax来实现模块的更替。
我们都知道，a标签的锚点值hash可以改变，但是页面却不会跳转。这就避免了浏览器请求页面的操作。我们可以结合锚点值的不同，通过ajax来进行异步数据请求，从而加载页面。也可以动态增加script或link标签，来增加需要的新代码块。

//监听锚点hash变动事件，获取实时的hash代码。
window.addEventListener('hashchange',function () {
    console.log(location.hash)
}
解决问题
  • 重复请求，重复编译相同代码块。
  • 可以记录用户请求记录。
