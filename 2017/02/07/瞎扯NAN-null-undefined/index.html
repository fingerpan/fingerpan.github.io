瞎扯NAN,null,undefined

发表于 2017-02-07 | 分类于 javascripte |

NAN,null,undefined 存在的意义。

typeof null       // 'object';
null === null     // true;
null === {}       //false;

查询null的类型是object。可以说null类型就是object吗？
其实也可以这么说吧，但有有点牵强。
我个人的观点是： null 是一个特殊的，特定的对象，也就是说内存地址是固定的对象。用来形容一个空/非对象数据类型。
这里的非对象并不是指简单类型。也不是等于一个{};

在javascript中，这三个特殊的值，初学者一直都不知道这三个东西到底又什么用。在这里，我想简单的说一下这三个东西存在的意义。

Undefined，顾名思义，就是“找不到”的意思。在Javascript中，它是一种数据类型，这个数据类型只有一个值，undefined。可以很通俗的说，这种数据类型是三大数据类型的一种补充，也是变量的一种状态描述。
为什么说是变量的一种状态描述呢，很好理解，从它的意思入手“没有找到”。javascript是一种弱类型的语言，与强类型语言的最大区别就是，当我们在声明变量的时候，并不需要对变量的类型进行声明。
比如说，我们声明一个变量a ;
在javascript我们可以这么声明
var a;
但是如果在java等其他强类型语言，我们必须类型声明。
声明一个整数型的变量a : int a;
声明一个字符串类型的a: char a;
所以，当我们想要知道一个为赋值a 的时候，在强类型的语言中我们可以准确获得，因为我们已经声明了类型。但是在javascript中，我们并不能知道，因为我们只是声明了变量，也没有赋值。
