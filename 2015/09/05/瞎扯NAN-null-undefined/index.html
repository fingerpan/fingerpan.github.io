瞎扯NAN,null,undefined

发表于 2015-09-05 | 分类于 javascript |
NaN,null,undefined 到底干啥子的。

太极讲阴阳，八卦论黑白，而机器，则是0和1之间的博弈。机器硬件要运行，离不开软件编程语言这个软组织。命令只有是与否，真与假之分。从这点上。任何语言在设计上都遵循这一点。Brendan Eich只花费了1周的时间就打造出了js，却也离不开这一点。NaN,null,undefined都是这个逻辑的体现。
今天讲一讲我理解的 NaN,null,undefined;这三个类型值的作用.

我们先看看这三个的解释。

1. undefined

Undefined，顾名思义，就是“找不到”的意思。在Javascript中，它是一种数据类型，这个数据类型只有一个值————undefined。

可以很通俗的说，这种数据类型是三大数据类型的一种补充，也是变量的一种状态描述。为什么说是变量的一种状态描述，很好理解，从它的意思入手——“没有找到/未定义”。javascript是一种弱类型的语言，与强类型语言的最大区别就是，当我们在声明变量的时候，并不需要对变量的类型进行声明。比如说，我们声明一个变量a ;在javascript我们可以这么声明

var a;

但是如果在java等其他强类型语言，我们必须类型声明。

声明一个整数型的变量a :   int  a;   // a 的类型是整数
声明一个字符串类型的a:    char a;   // a 的数据类型是字符串

所以，当我们想要知道变量a类型的时候，在强类型的语言中我们可以准确获得，因为我们已经声明了类型。但是在javascript中，我们并不能知道，因为我们只是声明了变量，也没有赋值。怎么办？js使用一个新的数据类型来对这种情况进行赋值表示，也就是我们的undefined。所以，我一直这么理解undefined

  • undefined 是一种数据类型，表示变量未定义或未赋值，或者对象的属性和方法为未定义。

现在我们想想，到底这个undefined 存在有什么意义？如果吗没有它会这么样？

  • 如果没有undefined，那么我们typeof一个没有赋值的变量a, a到底显示什么？ 报错？ 还是..？
  • 如果我们引用一个为定义的对象属性？ 我们也会选择报错吗？ 如果你是要语言设计作者，你会这么设计？

2. NaN

先来看看一些有趣的

我们都知道，js中有隐性转换这个特性，而隐性转换的成功率到底要为多少？没有疑问，js设计成百分之百，如果不是百分之百，那么选择报错也是跟尴尬的事情。js追求要弱类型，要灵活，那就彻底一点的灵活。
可以说,NaN也是为了这个而服务的。它代表着一个非数字。如果一个数据实在找不到可行的转换规则，那么就直接返回一个NaN。这个特殊的值，可以减少很多报错。

要注意: NaN === NaN 这个表达式为false,

3. null

同样的，我们看看一下查询结果：

typeof null       // 'object';
null === null     // true;
null === {}       //false;
  • 查询null的类型是object。直接说null类型就是object么？
    其实也可以这么说吧，但有有点牵强。我个人的观点是： null 是一个特殊的，特定的对象，也就是说内存地址是固定的对象。用来形容一个空/非对象数据类型。这里的非对象并不是指简单类型。也不是等于一个{};
